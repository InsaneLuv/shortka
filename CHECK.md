# Shortka
## Плюсы

- MVP за 1 час
- Минимальный набор зависимостей
- Приемлемая произовдительность
- Можно запустить на чём угодно, что может провайдить нам домен

## Минусы

Избыточность(или скорее нехватка) архитектурных решений, типа **СИНХРОННОЙ** базы данных sqlite3, для этой задачи есть намного более удачные решения по типу:
- MongoDB
- Redis

Для обеспечения баланса между надёжностью и производительностью я бы выбрал паттерн flight pattern.
При этом в качестве основного хранилища данных можно использовать любую из вышеупомянутых решений.

## Что не готово

1. Предполагались транзакции, но это слишком долго
2. Можно было бы заменить uvicorn логгер на struclog полностью
3. Больше логов
4. Дополнительные тесты, по типу тестов сервисов отдельно от апи

## Развитие

1. Добавил бы бонусные возможности из тз, но счёл это излишним в этом кейсе
2. Заменил sqlite на распределённый keyDB или чистый redis
